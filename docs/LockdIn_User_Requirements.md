| **ID** | **Requirement**                                                                              | **Category** | **Acceptance Criteria**                                                                  |
|--------|----------------------------------------------------------------------------------------------|--------------|------------------------------------------------------------------------------------------|
| HLR-1  | The system shall monitor daily and weekly app usage and display totals and trends.           | User         | After 24 h, a user sees total screen time and app-category breakdown on a dashboard.     |
| HLR-2  | The system shall let users define customizable lockdown rules (Android: non-bypassable).     | User         | When a limit is reached, configured apps become blocked and the UI shows a lock state.   |
| HLR-3  | The system shall provide behavioral insights (time of day, app categories, location nudges). | User         | Trends view highlights peak usage hours and suggests nudges for flagged times/locations. |
| HLR-4  | The system shall support multiple notification tone profiles (fun, edgy, professional).      | User         | Switching tone in Settings updates the copy used in the next reminder.                   |
| HLR-5  | The system shall offer accountability notifications to selected contacts.                    | User         | When a daily limit is exceeded, a pre-approved contact receives a notification summary.  |
| HLR-6  | The system shall include a quick onboarding flow completed in ≤ 2 minutes.                   | User         | First-run setup (permissions + default rule) finishes within 120 seconds.                |
| HLR-7  | The application shall be accessible to users with disabilities (WCAG 2.1).                   | Regulatory   | All interactive controls meet contrast and focus accessibility guidelines.               |
| HLR-8  | The product shall help users reduce average daily screen time by ≥ 25 % after 4 weeks.       | Business     | Pilot data shows ≥ 25 % median reduction from baseline.                                  |
| HLR-9  | At least 70 % of pilot users shall remain active after testing.                              | Business     | Retention metric ≥ 70 % week-over-week.                                                  |
| HLR-10 | At least 80 % of users shall view analytics weekly.                                          | Business     | Weekly analytics view rate ≥ 80 %.                                                       |
| HLR-11 | At least 50 % of users shall enable accountability notifications.                            | Business     | Feature enablement ≥ 50 % during pilot.                                                  |
| HLR-12 | User satisfaction shall average ≥ 4.0 / 5 on usefulness and ease of use.                     | Business     | Post-survey average ≥ 4.0.                                                               |
| HLR-13 | Android build shall enforce non-bypassable lockdown using permitted OS APIs.                 | Technical    | Apps lock correctly when limits are reached.                                             |
| HLR-14 | iOS build shall implement soft limits via Apple Screen Time/DeviceActivity APIs.             | Technical    | iOS shows reminders only; no OS-level blocking attempted.                                |
| HLR-15 | The system shall store minimal data and request explicit consent for permissions.            | Technical    | First-run prompts show data-use notice and collect consent.                              |
| HLR-16 | The app shall maintain \< 5 % crash rate across supported devices.                           | Technical    | Crash analytics show ≥ 95 % crash-free sessions.                                         |
| HLR-17 | The system shall comply with privacy and data-retention policies.                            | Regulatory   | Consent UX visible; retention configured; privacy statement published.                   |
| HLR-18 | The app shall meet accessibility standards (WCAG 2.1 AA).                                    | Regulatory   | Accessibility audits pass for contrast, labels, and focus.                               |
